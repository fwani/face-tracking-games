## 1. 📌 제품 개요

**제품명 (가칭)**  
장애물 피하기

**한 줄 설명**  
얼굴 움직임으로 날아오는 장애물을 피하는 카메라 기반 회피 게임

**핵심 컨셉**
- 카메라 기반 비접촉 게임
- 얼굴(고개 움직임)만으로 조작
- 직관적인 회피 중심 플레이
- 리듬/패턴 기반 몰입감

## 4. 🎮 게임 플레이

### 4.1 기본 구조
- 플레이어는 화면 **하단 근처**에서 **연속 이동**(yaw/pitch → 정규화 x,y)
- 장애물은 **위에서 아래로 낙하**(닷지/버릿헬 스타일)
- 얼굴을 움직여 장애물을 회피

---

### 4.2 입력 방식

| 입력 | 동작 |
|------|------|
| 고개 좌 | 왼쪽 이동 |
| 고개 우 | 오른쪽 이동 |
| 고개 숙이기 | 아래 회피 |
| 고개 들기 | 위 회피 |

---

### 4.3 입력 데이터 (ARKit)

- `yaw` → 좌우 이동
- `pitch` → 위/아래 이동

---

### 4.4 입력·플레이어 위치

- 기준 자세 대비 **yaw / pitch**(라디안)를 사용한다.
- 플레이어는 화면 내 **연속 좌표**로 이동하며, 축별 **dead zone**으로 미세 떨림을 줄인다(해당 축은 이전 프레임 위치 유지).
- 구현 세부: `ObstacleDodgeAnalogMapper`로 스무딩·클램프.

---

## 5. 🧠 시스템 구조

### 5.1 입력 파이프라인

```
ARKit Face Tracking
   ↓
ARFaceAnchor
   ↓
headTransform → yaw / pitch (기준 자세 보정 후 상대값)
   ↓
AnalogMapper (축별 dead zone, 스무딩)
   ↓
플레이어 정규화 위치 (x, y)
```

---

### 5.2 입력 처리

- 축별 dead zone: 해당 축 각도가 임계 안이면 **그 축 좌표는 유지**
- dead zone 밖이면 yaw→수평 오프셋, pitch→수직 오프셋으로 목표점 계산 후 클램프·스무딩

---

## 6. 🧱 장애물 시스템

### 6.1 장애물

- 원형 히트박스, **상단 밖에서 스폰** 후 **수직 낙하**
- 시각적 구분을 위한 팔레트(색)만 두고, 회피는 **플레이어 이동**으로 처리

---

### 6.2 패턴 시스템

- 난이도에 따라 스폰 **가로 위치**(랜덤 / 열 시퀀스 / 리듬형 밀집 간격) 조합
- 등장 간격·낙하 속도는 난이도 테이블로 조정

---

## 7. 🎯 충돌 시스템
