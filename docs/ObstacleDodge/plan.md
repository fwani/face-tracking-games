# ObstacleDodge 개발 계획

> 기준 문서: `docs/ObstacleDodge/prd.md`  
> 태스크는 체크박스로 진행 상태를 관리한다. (구현: `CameraGameHub/Games/ObstacleDodge/`)

## 프로젝트 개요

- **제품:** 얼굴(고개) 움직임만으로 날아오는 장애물을 피하는 카메라 기반 회피 게임.
- **입력:** ARKit `ARFaceAnchor` → 기준 자세 대비 **yaw / pitch** → `ObstacleDodgeAnalogMapper`로 **연속 정규화 위치 (x,y)** (축별 dead zone·스무딩).
- **플레이:** 플레이어는 화면 하단 근처에서 자유 이동, 장애물은 **위에서 아래로 낙하**(닷지 스타일). 충돌 시 즉시 게임오버.
- **MVP 성공 기준:** 회피 가능, 낮은 입력 지연, 1판 완주, 초보자 10초 내 조작 이해, 기본 UI·점수.

## Phase 1 — 프로젝트 골격·ARKit·yaw/pitch 추출

**목표:** 카메라 권한·Face Tracking 세션을 연결하고, PRD §4.3·§5.1에 따라 `headTransform` 기반 **yaw·pitch 수치**를 안정적으로 얻는다.

### 기타 시스템

- [x] ObstacleDodge 전용 진입 루트(루트 뷰 또는 코디네이터) 파일·모듈 경계 정의
- [x] `Info.plist`에 카메라 사용 목적 문자열(`NSCameraUsageDescription`) 추가 — 앱 타깃 `INFOPLIST_KEY_NSCameraUsageDescription`로 반영됨
- [x] 앱 실행 시 카메라 권한 요청 및 거부 시 사용자 안내(문구·화면) 정의

### 입력 처리 (Face Tracking)

- [x] `ARSession`에 Face Tracking 구성(`ARFaceTrackingConfiguration` 등) 적용
- [x] `ARFaceAnchor` 갱신 콜백에서 head pose 행렬 접근 지점 확정
- [x] head pose로부터 **yaw** 각도 산출(부호: 좌/우 방향 규칙 문서화)
- [x] head pose로부터 **pitch** 각도 산출(부호: 위/아래 방향 규칙 문서화)
- [x] yaw/pitch 단위(라디안/도)와 표시·로직 내부 표현을 한 가지로 통일
