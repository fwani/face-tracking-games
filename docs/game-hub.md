# Camera Game Hub (카메라 게임 허브)

**전면·후면 등 카메라(및 필요 시 트래킹)로 몸·얼굴 등을 읽어 게임에 쓰는** 미니게임을 한 앱에서 고르고 실행하는 구조를 목표로 한다. 제품명은 **Camera Game Hub**로 부른다.

### 이름에 대해

말 그대로 화면 너머 현실 위에 오브젝트를 겹쳐 보는 **「증강현실 게임」**과는 결이 다를 수 있다. 구현에서는 Apple의 **ARKit**(예: 얼굴 트래킹)을 쓸 수 있지만, 사용자에게는 **비터치·카메라 기반 미니게임**으로 소개하는 편이 맞다. 이후 다른 카메라·트래킹 조합을 넣어도 같은 허브 아래에 둘 수 있다.

## 목표

단일 게임이 아니라, **첫 화면에서 게임을 고르고** 선택한 게임만 실행되는 앱으로 확장한다. 지금 구현된 FaceFly는 **얼굴 트래킹 입력**을 쓰는 미니게임으로, 그중 하나의 타이틀로 유지한다.

## 사용자 흐름

1. **Camera Game Hub(앱 루트, 게임 선택)**  
   여러 게임 카드·목록 등으로 진입점을 제공한다.

2. **FaceFly 선택 시**  
   현재 앱에서 보이는 FaceFly 전용 홈(난이도·시작 등)으로 이동하고, 기존과 같이 플레이한다.

3. **다른 게임 추가 시**  
   동일한 선택 화면에서 새 항목을 추가하고, 탭/네비게이션으로 **해당 게임 전용 루트 뷰**로 진입한다. 각 게임은 자체 입력 파이프라인·홈·루프·에셋을 가질 수 있다.

## 구현 방향(권장)

- **프로젝트:** Xcode 프로젝트·타깃·생성 앱 이름은 **CameraGameHub** (`CameraGameHub.xcodeproj`, 폴더 `CameraGameHub/`).
- **루트:** 앱에서는 `ContentView`의 `AppRoot`(`.hub` / `.faceFly`)로 분기하고, 허브 화면은 `CameraGameHubView`를 둔다. 게임 카드·추가 진입은 여기에 이어 붙이면 된다.
- **FaceFly:** 기존 `HomeView` → 얼굴 트래킹/게임 플로우는 현재 모듈 구조를 그대로 “한 미니게임 모듈”로 묶는다. FaceFly 홈 상단의 **게임 선택**으로 허브로 돌아온다.
- **추후 게임:** `FaceFly`와 동일한 패턴으로 폴더(`Games/SomeGame/`) 등을 두고, 게임별로 필요한 세션·`ARConfiguration`(또는 다른 입력 스택)만 다르게 가져간다. 루트에서는 진입만 연결한다.

세부 PRD·태스크는 게임별로 `docs/facefly/` 등 폴더에 둔다.
